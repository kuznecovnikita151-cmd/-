1. Краткое описание проекта :
 1.1. Ф.И.О. разработчиков. Кузнецов Никита Владимирович 
 1.2. Группа. ЛПЦ24-ОБ-ПМ1
 1.3. Тема проекта. «Проектирование и разработка прототипа интерактивного образовательного веб-портала для пациентов с впервые выявленным сахарным диабетом 1 типа»
 1.4. Цель проекта. Спроектировать и разработать функциональный прототип интерактивного образовательного веб-портала для ускоренного формирования жизненно важных практических навыков самоконтроля и снижения уровня тревожности у пациентов с впервые выявленным сахарным диабетом 1 типа.
 1.5. Задачи проекта. 
1.  Спроектировать и реализовать структуру из четырех базовых образовательных модулей («Что такое диабет?», «Питание», «Физическая активность», «Самоконтроль») с интерактивными элементами внутри каждого.
2.  Разработать и интегрировать в портал три ключевых инструмента: калькулятор хлебных единиц (ХЕ) с базой продуктов, систему проверочных тестов с обратной связью и электронный дневник самочувствия для записи показателей глюкозы и субъективных ощущений.
3.  Создать личный кабинет пользователя, обеспечивающий сохранение прогресса обучения, результатов тестирования и данных из дневника, а также предоставить адаптивный интерфейс для работы на различных устройствах.
4.  Провести функциональное и юзабилити-тестирование прототипа, оценив корректность работы инструментов и удобство навигации по образовательным модулям, а также подготовить документацию по проекту.
 1.6. Аннотация. Данный проект направлен на решение проблемы информационного кризиса и дезадаптации пациентов после манифестации сахарного диабета 1 типа (СД1). В ходе проекта будет создан сайт, структурированный в виде пошаговых обучающих модулей по ключевым темам: «Что такое диабет?», «Питание», «Физическая активность» и «Самоконтроль». Портал будет оснащён интерактивными практическими инструментами: калькулятором хлебных единиц (ХЕ) для расчёта доз инсулина, системой проверочных тестов и электронным дневником самочувствия для фиксации показателей глюкозы.
2. Предобработка данных (приведение данных к единому формату, устранение пропусков в данных, удаление дубликатов, аномалий).
3. Визуализация данных.
Проект визуализируется как единый веб-интерфейс, центром которого является личный кабинет пользователя с панелью прогресса. От неё осуществляется доступ к линейной цепочке из четырёх интерактивных образовательных модулей и к отдельной панели инструментов, содержащей калькулятор ХЕ, форму дневника самочувствия и раздел с тестами. Все элементы будут выполнены в адаптивном, минималистичном дизайне с акцентом на интуитивную навигацию и визуальную чёткость.
4. Корреляционный анализ. 
В рамках проекта корреляционный анализ будет направлен на исследование взаимосвязей между данными, собираемыми в системе. На этапе разработки дневника самочувствия будет спроектирована структура данных, позволяющая в дальнейшем анализировать статистические зависимости между ключевыми параметрами: количеством потребленных хлебных единиц (рассчитанных в калькуляторе ХЕ), уровнем глюкозы крови (до и после приема пищи), введенной дозой инсулина и физической активностью. Цель анализа — выявление потенциальных индивидуальных паттернов и триггеров скачков глюкозы, что теоретически позволит порталу генерировать персонализированные гипотезы для пользователя (например, «После потребления X ХЕ ваш сахар повышается в среднем на Y ммоль/л»). Этот функционал закладывается как перспективный для будущего развития системы.
5. Регрессионный анализ (линейная регрессия, множественная линейная регрессия, ...).
На основе данных, аккумулируемых в дневнике самочувствия, в перспективе может быть применен регрессионный анализ. Ключевым методом рассматривается **множественная линейная регрессия**, где уровень постпрандиальной (после еды) глюкозы выступает в роли зависимой переменной, а независимыми предикторами становятся количество хлебных единиц (из калькулятора ХЕ), введённая доза болюсного инсулина, уровень глюкозы перед едой и длительность физической активности. Построение такой модели позволит количественно оценить вклад каждого фактора в изменение сахара крови, предоставив пользователю не просто констатацию связи, а инструмент для более точного прогнозирования. Этот аналитический модуль проектируется как основа для будущей интеллектуальной системы поддержки принятия решений в рамках портала.
